World Generation

  • Languages: Java
  • Github URL: Project Link

Created a playable game in Java that requires the player to input movement with their keyboard to navigate and escape a world. I also wrote a World-Generation algorithm to pseudorandomly create rooms/hallways/features for that world. This project was for UC Berkeley's CS61B course.

Uses Java's Random package to pseudorandomly generate values from a user-inputed seed. These values were used to determine the number, size, and position of the rooms and hallways within the world. They were also used to place the interactable key and door which the player needed to find in order to escape. The game also supports saving or loading the state of the world, including any changes the player made to it. The program I also added the ability for the player to toggle a lightswitch to illuminate the entire world rather than a smaller area around the player.

I used Java HashMaps to store data for each tile (its position, identity, state in the game, etc), and used sets to keep track of conditions that needed to be met during world generation (like the connectedness of all rooms). I implemented a Disjoint Set system to manage the sets, and had some unique uses of datastructures to keep world generation efficient but robust.
